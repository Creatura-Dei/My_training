МЕТОДЫ ПРИМИТИВОВ JS позволяет работать с примитивами как с объектами НО Они НЕ объекты !!! РАЗЛИЧИЯ: Примитив: 1) Есть 7 примитивных типов: string, number, boolean, symbol, null, undefined, bigint Объект: 1) Хранит множество значений (свойств) 2) Объявление через {} или же функции (тоже объекты) 3) Хранение функции как свойство - метод _______________________________________________________________________________ ПРИМИТИВ КАК ОБЪЕКТ Чтобы это работало, при такое доступе создаётся специальный "ОБЪЕКТ-ОБЁРТКА", который исполняет функциональность, а после удаляется. Каждый примитив имеет свой собственный "объект-обёртку" (с соответствующими методами): String ( например: str.toUpperCase() ); Number ( например: num.toFixed(2) ) Boolean Symbol BigInt НЕ ИМЕЮТ МЕТОДОВ: null undefined Пример работы: let str = "Привет"; alert( str.toUpperCase() ); // ПРИВЕТ 1) Строка str - примитив. В момент обращения к его свойству, создаётся специальный объект, который знает строки и имеет такие полезные методы, как toUpperCase(). 2) Этот метод запускается и возвращает новую строку (в alert) 3) Специальный объект удаляется, оставля только примитив str ТО ЕСТЬ Примитивы могут предоставлять методы, и в то же время оставаться "лёгкими" _______________________________________________________________________________ !!!!!!!!!!!!!!!!!!!!!!!!!!!!! КОНСТРУКТОРЫ String/Number/Boolean предназначены только для внутреннего пользования !!!!!!!!!!!!!!!!!!!!!!!!!!!!! Явное создание "объектов-обёрток" для примитивов при помощи такого синтаксиса как new Number(1) или new Boolean(false).
